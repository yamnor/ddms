## この講座の進め方

- 今回の講習会には、「実験は専門だが第一原理計算はこれから」という方から、「シミュレーションは得意だがデータ駆動型材料開発は初めて」という方まで、**スタート地点の異なる参加者**のみなさんが集まることを想定しています
- そのため、本講座では、第一原理計算そのものの理論や、特定のソフトウェアの細かい操作方法を深掘りするのではなく、**「データをつくる装置（Data Generator）」として第一原理計算を安全・安心に使いこなすための「取り扱い説明（インストラクション）」** に焦点を当てて進めます
- 実習では、`ASE` + `GPAW` を用いた Python コードを紹介しますが、コードの一行一行を暗記したり、Python 言語そのものを深く理解することはゴールではありません
	- 中身のコードは「装置の内部機構」として位置づけ、**ユーザーとしてどのような設定（カットオフ、k点、スキャン範囲など）で使うと、安全で再現性の高い運用になるのか** を体験的に理解することを重視します
- この講座で、「第一原理計算を用いてデータをつくるときに押さえておきたい基本部品（`ASE` と `GPAW`、Colab 上の計算環境など）」の役割がつかめれば、あとは他のソフトウェアやワークフローにも応用が利くはずです
- 特に近年は、Google Colab 上でも AI アシスタントを活用できるようになってきていますので、「どのライブラリを組み合わせて、どのような条件で計算させれば良いか」という**設計方針さえ理解できれば、AI にコード生成を手伝ってもらいながら、自分の目的に合ったデータ生成ワークフローを組み立てていくこと**も十分に可能です

## この講座の到達目標

- データ駆動型材料開発の流れの中で、**第一原理計算が「データをつくる装置（Data Generator）」として果たす役割**を、自分の言葉で説明できる
- `ASE` + `GPAW` を代表例として、**クラウド環境（Google Colab）上に第一原理計算の場を用意し、再現性のある条件で「シリコンの Volume–Energy データ」を自動的に集める一連の手順**を、概念レベルで説明できる
- 実習で扱う Python コードの細部を暗記していなくても、**ユーザーとしてどのような計算条件（カットオフ、k点、スキャン範囲など）を指定すれば、安全・安心で物理的に妥当な計算になるか**を判断するための基本的な観点を理解できる
- 第一原理計算で生成したデータを、**状態方程式フィッティングや CSV への保存を通じて機械学習モデルに渡し、「計算でデータをつくり、それを機械学習で活用する」一連のワークフロー**のイメージを持てる
- こうしたワークフローと部品（`ASE`、`GPAW`、Colab、AI アシスタントなど）の役割が整理できれば、**他のソフトウェアや別の物性・材料系に対しても、自分なりに「データをつくる装置」を設計・拡張していける**という見通しを持てる
